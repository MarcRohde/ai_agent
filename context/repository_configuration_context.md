# Repository Configuration Context

## Header

Purpose: Standardize how repositories are configured for layered agent memory and note-taking loops.
Use when: Bootstrapping a new repository or upgrading an existing one to this model.

## Content

- Layer 1 is global profile memory in agent settings and should stay concise.
- Layer 2 is repository mission memory via `AGENTS.md` and `CLAUDE.md` at repo root.
- Layer 3 is repository context memory via `context/` files with reusable standards and examples.
- Learning logs are committed in git to preserve traceability and compounding quality.
- Mission files should include Purpose, Tree, Rules, and Note-Taking Loop sections.
- Context files should include Header, Content, and Learning Log sections.
- When 3 or more similar lessons appear, create a focused context file and update indexes.
